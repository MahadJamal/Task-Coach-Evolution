  1  ''' 
  2  Base classes and functions related to listening of pubsub messages. 
  3  Listeners are callable objects that get subscribed to a pubsub 
  4  topic.  
  5   
  6  Each instance of Listener holds its callable listener only by weak 
  7  reference so it doesn't prevent the callable from being garbage collected 
  8  when callable is no longer in use by the application. A Listener instance  
  9  has the same hash value as the callable that it wraps.  
 10   
 11  A callable will be given data when a message is sent to it. The way this  
 12  data is packaged is the "message protocol". For example in the kwargs  
 13  protocol the data is sent via keyword arguments, whereas in the arg1  
 14  protocol it is sent as fields of a Message object.  
 15   
 16  In order for a listener to subscribe to a topic, it must be capable of  
 17  receiving this data. If pubsub determines that this is not the case, it 
 18  will raise a ListenerInadequate exception. This check uses the topic's  
 19  "message protocol specification" (MPS). For 
 20  instance, if topic A has an LPS "arg1, arg2=None"", then only listeners 
 21  of the form callable([self,] arg1, arg2=something) will be accepted. 
 22   
 23  Since MPS verification depends on the messaging protocol, only the  
 24  base capability is defined in this module; actual validators are  
 25  defined in the protocol folders (kwargs and arg1) by deriving from  
 26  the base class.  
 27

 30
