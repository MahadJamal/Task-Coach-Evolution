Class TreeCtrlDragAndDropMixin

source code

     object --+    
              |    
TreeHelperMixin --+
                  |
                 TreeCtrlDragAndDropMixin

This is a mixin class that can be used to easily implement
dragging and dropping of tree items. It can be mixed in with 
wx.TreeCtrl, wx.gizmos.TreeListCtrl, or wx.lib.customtree.CustomTreeCtrl.

To use it derive a new class from this class and one of the tree
controls, e.g.:
class MyTree(TreeCtrlDragAndDropMixin, wx.TreeCtrl):
    ...

You *must* implement OnDrop. OnDrop is called when the user has
dropped an item on top of another item. It's up to you to decide how
to handle the drop. If you are using this mixin together with the
VirtualTree mixin, it makes sense to rearrange your underlying data
and then call RefreshItems to let the virtual tree refresh itself.

OnDrop(self, dropItem, dragItems, part)

source code 

This function must be overloaded in the derived class. dragItems are the items being dragged by the user. dropItem is the item the dragItems are dropped on. If the user doesn't drop the dragItems on another item, dropItem equals the (hidden) root item of the tree control. `part` is 0 if the items were dropped on the middle third of the dropItem, -1 if they were dropped on the upper third and 1 for the lower third.

OnBeginDrag(self, event)

source code 

This method is called when the drag starts. It either allows the drag and starts it or it vetoes the drag when the the root item is one of the dragged items.
