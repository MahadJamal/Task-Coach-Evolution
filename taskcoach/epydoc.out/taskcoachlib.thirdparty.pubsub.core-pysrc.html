 1  '''
 
 2  Core package of pubsub, holding the publisher, listener, and topic
 
 3  object modules. Functions defined here are used internally by
 
 4  pubsub so that the right modules can be found later, based on the
 
 5  selected messaging protocol.
 
 6  
 
 7  Indeed some of the API depends on the messaging
 
 8  protocol used. For instance sendMessage(), defined in publisher.py,
 
 9  has a different signature (and hence implementation) for the kwargs
 
10  protocol than for the arg1 protocol.
 
11  
 
12  The most convenient way to
 
13  support this is to put the parts of the package that differ based
 
14  on protocol in separate folders, and add one of those folders to
 
15  the package's __path__ variable (defined automatically by the Python
 
16  interpreter when __init__.py is executed). For instance, code
 
17  specific to the kwargs protocol goes in the kwargs folder, and code
 
18  specific to the arg1 protocol in the arg1 folder. Then when doing
 
19  "from pubsub.core import listener", the correct listener.py will be
 
20  found for the specified protocol. The default protocol is kwargs.
 
21  
 
22  Only one protocol can be used in an application. The default protocol,
 
23  if none is chosen by user, is kwargs, as selected by the call to
 
24  _prependModulePath() at end of this file.
