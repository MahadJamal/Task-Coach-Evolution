Source Code for Package taskcoachlib.thirdparty.aui

  1  """ 
  2  AUI is an Advanced User Interface library that aims to implement "cutting-edge" 
  3  interface usability and design features so developers can quickly and easily create 
  4  beautiful and usable application interfaces. 
  5   
  6   
  7  Vision and Design Principles 
  8  ============================ 
  9   
 10  AUI attempts to encapsulate the following aspects of the user interface: 
 11   
 12  * **Frame Management**: Frame management provides the means to open, move and hide common 
 13    controls that are needed to interact with the document, and allow these configurations 
 14    to be saved into different perspectives and loaded at a later time.  
 15   
 16  * **Toolbars**: Toolbars are a specialized subset of the frame management system and should 
 17    behave similarly to other docked components. However, they also require additional 
 18    functionality, such as "spring-loaded" rebar support, "chevron" buttons and end-user 
 19    customizability.  
 20   
 21  * **Modeless Controls**: Modeless controls expose a tool palette or set of options that 
 22    float above the application content while allowing it to be accessed. Usually accessed 
 23    by the toolbar, these controls disappear when an option is selected, but may also be 
 24    "torn off" the toolbar into a floating frame of their own.  
 25   
 26  * **Look and Feel**: Look and feel encompasses the way controls are drawn, both when shown 
 27    statically as well as when they are being moved. This aspect of user interface design 
 28    incorporates "special effects" such as transparent window dragging as well as frame animation.  
 29   
 30  AUI adheres to the following principles: 
 31   
 32  - Use native floating frames to obtain a native look and feel for all platforms; 
 33  - Use existing wxPython code where possible, such as sizer implementation for frame management;  
 34  - Use standard wxPython coding conventions. 
 35   
 36   
 37  Usage 
 38  ===== 
 39   
 40  The following example shows a simple implementation that uses :class:`framemanager.AuiManager` to manage 
 41  three text controls in a frame window:: 
 42   
 43      import wx 
 44      import wx.lib.agw.aui as aui 
 45   
 46      class MyFrame(wx.Frame): 
 47   
 48          def __init__(self, parent, id=-1, title="AUI Test", pos=wx.DefaultPosition, 
 49                       size=(800, 600), style=wx.DEFAULT_FRAME_STYLE): 
 50   
 51              wx.Frame.__init__(self, parent, id, title, pos, size, style) 
 52   
 53              self._mgr = aui.AuiManager() 
 54               
 55              # notify AUI which frame to use 
 56              self._mgr.SetManagedWindow(self) 
 57   
 58              # create several text controls 
 59              text1 = wx.TextCtrl(self, -1, "Pane 1 - sample text", 
 60                                  wx.DefaultPosition, wx.Size(200,150), 
 61                                  wx.NO_BORDER | wx.TE_MULTILINE) 
 62                                                  
 63              text2 = wx.TextCtrl(self, -1, "Pane 2 - sample text", 
 64                                  wx.DefaultPosition, wx.Size(200,150), 
 65                                  wx.NO_BORDER | wx.TE_MULTILINE) 
 66                                                  
 67              text3 = wx.TextCtrl(self, -1, "Main content window", 
 68                                  wx.DefaultPosition, wx.Size(200,150), 
 69                                  wx.NO_BORDER | wx.TE_MULTILINE) 
 70               
 71              # add the panes to the manager 
 72              self._mgr.AddPane(text1, AuiPaneInfo().Left().Caption("Pane Number One")) 
 73              self._mgr.AddPane(text2, AuiPaneInfo().Bottom().Caption("Pane Number Two")) 
 74              self._mgr.AddPane(text3, AuiPaneInfo().CenterPane()) 
 75                                     
 76              # tell the manager to "commit" all the changes just made 
 77              self._mgr.Update() 
 78   
 79              self.Bind(wx.EVT_CLOSE, self.OnClose) 
 80   
 81   
 82          def OnClose(self, event): 
 83   
 84              # deinitialize the frame manager 
 85              self._mgr.UnInit() 
 86   
 87              self.Destroy()         
 88              event.Skip()         
 89   
 90   
 91      # our normal wxApp-derived class, as usual 
 92   
 93      app = wx.App(0) 
 94   
 95      frame = MyFrame(None) 
 96      app.SetTopWindow(frame) 
 97      frame.Show() 
 98   
 99      app.MainLoop()

136  Plus the following features: 
137   
138  - AuiManager: 
139   
140    (a) Implementation of a simple minimize pane system: Clicking on this minimize button causes a new 
141        AuiToolBar to be created and added to the frame manager, (currently the implementation is such 
142        that panes at West will have a toolbar at the right, panes at South will have toolbars at the 
143        bottom etc...) and the pane is hidden in the manager. 
144        Clicking on the restore button on the newly created toolbar will result in the toolbar being 
145        removed and the original pane being restored; 
146    (b) Panes can be docked on top of each other to form `AuiNotebooks`; `AuiNotebooks` tabs can be torn 
147        off to create floating panes; 
148    (c) On Windows XP, use the nice sash drawing provided by XP while dragging the sash; 
149    (d) Possibility to set an icon on docked panes; 
150    (e) Possibility to draw a sash visual grip, for enhanced visualization of sashes; 
151    (f) Implementation of a native docking art (`ModernDockArt`). Windows XP only, **requires** Mark Hammond's 
152        pywin32 package (winxptheme); 
153    (g) Possibility to set a transparency for floating panes (a la Paint .NET); 
154    (h) Snapping the main frame to the screen in any positin specified by horizontal and vertical 
155        alignments; 
156    (i) Snapping floating panes on left/right/top/bottom or any combination of directions, a la Winamp; 
157    (j) "Fly-out" floating panes, i.e. panes which show themselves only when the mouse hover them; 
158    (k) Ability to set custom bitmaps for pane buttons (close, maximize, etc...); 
159    (l) Implementation of the style ``AUI_MGR_ANIMATE_FRAMES``, which fade-out floating panes when 
160        they are closed (all platforms which support frames transparency) and show a moving rectangle 
161        when they are docked and minimized (Windows < Vista and GTK only); 
162    (m) A pane switcher dialog is available to cycle through existing AUI panes; 
163    (n) Some flags which allow to choose the orientation and the position of the minimized panes; 
164    (o) The functions [Get]MinimizeMode() in `AuiPaneInfo` which allow to set/get the flags described above; 
165    (p) Events like ``EVT_AUI_PANE_DOCKING``, ``EVT_AUI_PANE_DOCKED``, ``EVT_AUI_PANE_FLOATING`` and ``EVT_AUI_PANE_FLOATED`` are 
166        available for all panes *except* toolbar panes; 
167    (q) Implementation of the RequestUserAttention method for panes; 
168    (r) Ability to show the caption bar of docked panes on the left instead of on the top (with caption 
169        text rotated by 90 degrees then). This is similar to what `wxDockIt` did. To enable this feature on any 
170        given pane, simply call `CaptionVisible(True, left=True)`; 
171    (s) New Aero-style docking guides: you can enable them by using the `AuiManager` style ``AUI_MGR_AERO_DOCKING_GUIDES``; 
172    (t) A slide-in/slide-out preview of minimized panes can be seen by enabling the `AuiManager` style 
173        ``AUI_MGR_PREVIEW_MINIMIZED_PANES`` and by hovering with the mouse on the minimized pane toolbar tool; 
174    (u) New Whidbey-style docking guides: you can enable them by using the `AuiManager` style ``AUI_MGR_WHIDBEY_DOCKING_GUIDES``; 
175    (v) Native of custom-drawn mini frames can be used as floating panes, depending on the ``AUI_MGR_USE_NATIVE_MINIFRAMES`` style; 
176    (w) A "smooth docking effect" can be obtained by using the ``AUI_MGR_SMOOTH_DOCKING`` style (similar to PyQT docking style); 
177    (x) Implementation of "Movable" panes, i.e. a pane that is set as `Movable()` but not `Floatable()` can be dragged and docked 
178        into a new location but will not form a floating window in between. 
179     
180   
181  - AuiNotebook: 
182   
183    (a) Implementation of the style ``AUI_NB_HIDE_ON_SINGLE_TAB``, a la :mod:`lib.agw.flatnotebook`; 
184    (b) Implementation of the style ``AUI_NB_SMART_TABS``, a la :mod:`lib.agw.flatnotebook`; 
185    (c) Implementation of the style ``AUI_NB_USE_IMAGES_DROPDOWN``, which allows to show tab images 
186        on the tab dropdown menu instead of bare check menu items (a la :mod:`lib.agw.flatnotebook`); 
187    (d) 6 different tab arts are available, namely: 
188     
189        (1) Default "glossy" theme (as in :class:`~auibook.AuiNotebook`) 
190        (2) Simple theme (as in :class:`~auibook.AuiNotebook`) 
191        (3) Firefox 2 theme 
192        (4) Visual Studio 2003 theme (VC71) 
193        (5) Visual Studio 2005 theme (VC81) 
194        (6) Google Chrome theme 
195   
196    (e) Enabling/disabling tabs; 
197    (f) Setting the colour of the tab's text; 
198    (g) Implementation of the style ``AUI_NB_CLOSE_ON_TAB_LEFT``, which draws the tab close button on 
199        the left instead of on the right (a la Camino browser); 
200    (h) Ability to save and load perspectives in `AuiNotebook` (experimental); 
201    (i) Possibility to add custom buttons in the `AuiNotebook` tab area; 
202    (j) Implementation of the style ``AUI_NB_TAB_FLOAT``, which allows the floating of single tabs. 
203        Known limitation: when the notebook is more or less full screen, tabs cannot be dragged far 
204        enough outside of the notebook to become floating pages; 
205    (k) Implementation of the style ``AUI_NB_DRAW_DND_TAB`` (on by default), which draws an image 
206        representation of a tab while dragging; 
207    (l) Implementation of the `AuiNotebook` unsplit functionality, which unsplit a splitted AuiNotebook 
208        when double-clicking on a sash; 
209    (m) Possibility to hide all the tabs by calling `HideAllTAbs`; 
210    (n) wxPython controls can now be added inside page tabs by calling `AddControlToPage`, and they can be 
211        removed by calling `RemoveControlFromPage`; 
212    (o) Possibility to preview all the pages in a `AuiNotebook` (as thumbnails) by using the `NotebookPreview` 
213        method of `AuiNotebook`; 
214    (p) Tab labels can be edited by calling the `SetRenamable` method on a `AuiNotebook` page; 
215    (q) Support for multi-lines tab labels in `AuiNotebook`; 
216    (r) Support for setting minimum and maximum tab widths for fixed width tabs; 
217    (s) Implementation of the style ``AUI_NB_ORDER_BY_ACCESS``, which orders the tabs by last access time 
218        inside the Tab Navigator dialog; 
219    (t) Implementation of the style ``AUI_NB_NO_TAB_FOCUS``, allowing the developer not to draw the tab 
220        focus rectangle on tne `AuiNotebook` tabs. 
221   
222  | 
223   
224  - AuiToolBar: 
225   
226    (a) ``AUI_TB_PLAIN_BACKGROUND`` style that allows to easy setup a plain background to the AUI toolbar, 
227        without the need to override drawing methods. This style contrasts with the default behaviour 
228        of the :class:`~auibar.AuiToolBar` that draws a background gradient and this break the window design when 
229        putting it within a control that has margin between the borders and the toolbar (example: put 
230        :class:`~auibar.AuiToolBar` within a :class:`StaticBoxSizer` that has a plain background); 
231    (b) `AuiToolBar` allow item alignment: http://trac.wxwidgets.org/ticket/10174; 
232    (c) `AUIToolBar` `DrawButton()` improvement: http://trac.wxwidgets.org/ticket/10303; 
233    (d) `AuiToolBar` automatically assign new id for tools: http://trac.wxwidgets.org/ticket/10173; 
234    (e) `AuiToolBar` Allow right-click on any kind of button: http://trac.wxwidgets.org/ticket/10079; 
235    (f) `AuiToolBar` idle update only when visible: http://trac.wxwidgets.org/ticket/10075; 
236    (g) Ability of creating `AuiToolBar` tools with [counter]clockwise rotation. This allows to propose a 
237        variant of the minimizing functionality with a rotated button which keeps the caption of the pane 
238        as label; 
239    (h) Allow setting the alignment of all tools in a toolbar that is expanded; 
240    (i) Implementation of the ``AUI_MINIMIZE_POS_TOOLBAR`` flag, which allows to minimize a pane inside 
241        an existing toolbar. Limitation: if the minimized icon in the toolbar ends up in the overflowing 
242        items (i.e., a menu is needed to show the icon), this style will not work.
